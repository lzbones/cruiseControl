## 1. CC 决策算法原理

## 1. CC 决策算法原理

CC（Cruise Control，定速巡航控制）是一个人机共驾的低程度自动驾驶功能，它通过响应驾驶员指令来实施控制，使车辆能根据驾驶员的意图实现自动定速行驶。在 CC 这个人机共驾系统中，依然遵循人驾优先的原则。

决策算法基于驾驶人指令和系统状态进行模糊判定。

### 1.1. 驾驶员输入信号与指令映射
物理传感器和驾驶员输入信号通过以下逻辑映射到决策中：
* **按键输入**：增速键映射为 $I_0$，降速键映射为 $I_1$，取消键映射为 $I_2$，退出键映射为 $I_4$。
* **制动踏板踩下**：作为紧急人工干预输入，直接映射为 **刹车指令 ($I_3$)**。
* **人工油门踩下**：作为动力骑跨信号，体现在系统子态的 $b_1$ 位（$b_1=1$ 代表有油， $b_1=0$ 代表无油），影响系统状态的判定。例如，在待命状态下踩油门会使系统处于“有油待命 ($S_4$)”状态，此时增速/降速按键将触发无继控制（$R_5$）直接进入定速巡航。

当输入不同的驾驶指令 $I_m$ 时，系统根据当前的运行状态 $S_n$，决策出相应的响应动作 $R_k$（决策定义见下文 1.3 节）：

> [!NOTE]  
> **低速在控主动安全保护（新增）**：  
> 若系统处于 $S_6$（低速在控）状态，且持续时间超过阈值 $t_{timeout}$（推荐为 $3\text{s}$），即使无任何驾驶指令输入（`COMMAND_NONE`），系统也必须**主动决策并执行 $R_7$（报错待命）**，切断动力控制输出，以保护电机/发动机不受过载损伤。

### 1.3. 决策响应动作 $R_k$ 定义
系统输出的决策名称 $R_k$ 对应系统响应如下：
* $R_1$：**目标增量** — 定量增加设定目标车速。
* $R_2$：**目标减量** — 定量减少设定目标车速。
* $R_3$：**保持控制** — 维持当前目标车速不变，暂不做调节。
* $R_4$：**继承控制** — 进入控制，并载入上次有效的历史目标车速（即 Resume 逻辑）。
* $R_5$：**无继控制** — 进入控制，以当前实际车速作为新的目标车速（即 Set 逻辑）。
* $R_6$：**系统待命** — 退出控制，断开驱动扭矩，进入待命状态。
* $R_7$：**报错待命** — 退出控制，报车速异常错误，进入待命状态。
* $R_8$：**系统退出** — CC 功能彻底关闭。

## 2. 定速控制算法（SPPVT）原理

CC 系统的控制对象是车路系统，所以采用阶式惩罚型变目标比例控制算法（SPPVT，Stepwise Proportional control algorithm with Penalty Variable Target），以克服不断变动的车路系统阻力。该方法为无模型控制，无需车路系统模型参数。

### 2.1. 基本控制律
第 $i$ 级控制器的扭矩输出 $T_i(t)$ 为：
$$T_i(t) = K_p \Delta v_i(t) \quad (i = 1, 2, \dots) \tag{1}$$

其中，$K_p$ 为比例控制系数（单位：$\text{Nm}\cdot\text{s}/\text{m}$）。第 $i$ 级车速控制偏差 $\Delta v_i(t)$ 的计算表达式为：
$$\Delta v_i(t) = V_i - v_i(t) \tag{2}$$

式中，$V_i$ 为第 $i$ 级内部虚拟目标车速，$v_i(t)$ 为当前车辆实际速度。

### 2.2. 变目标判定条件 (修正项)
定义系统的**变目标时刻** $t_i$，即车辆速度已经在当前虚拟目标下稳定、加速度趋近于 0 的时刻。该时刻必须满足**双边绝对值限幅条件**：
$$|\dot{v}(t_i)| < \delta \tag{3}$$

其中，$\delta$ 为设定的变目标加速度阈值常量（单位：$\text{m/s}^2$，通常取 $[0.02, 0.05]$ 区间内的正小量）。

> [!IMPORTANT]  
> **防止连续触发的锁存/冷却机制（新增）**：  
> 在代码实现中，一旦在时刻 $t_i$ 触发了变目标更新，系统必须启动锁存或冷却机制（如设置冷却计时器 $T_{cooldown} \ge 2\text{s}$），在冷却时间内禁止连续触发式（3），以给车辆动力系统留出响应时间，避免虚拟目标车速发生高频阶跃积累导致振荡。

### 2.3. 目标车速更新律 (修正项)
在满足变目标时刻 $t_i$ 的条件时，更新下一级的虚拟目标车速 $V_{i+1}$：
$$V_{i+1} = V_i + \rho \Delta V_{ti} \quad (\text{初始状态 } V_1 = V_t) \tag{4}$$

式中，$\rho$ 为惩罚系数（常量，要求 $0 < \rho < 0.5$），$V_t$ 为驾驶员设定的 CC 目标车速。当前级与最终目标车速的偏差 $\Delta V_{ti}$ 为：
$$\Delta V_{ti} = V_t - v_i(t_i) \quad (\text{要求 } v_i(t) < V_t) \tag{5}$$

#### 2.3.1. 虚拟目标车速上限保护 (新增抗积分饱和机制)
为防止在超载、陡坡爬坡等极限阻力工况下，虚拟目标车速 $V_i$ 无限制增大导致翻越坡顶后车辆暴冲，必须对 $V_i$ 进行上限截断保护：
$$V_i \le V_t + V_{max\_offset} \tag{6}$$

其中 $V_{max\_offset}$ 是允许的最大虚拟增量（例如限制为设定车速 $V_t$ 的 $15\%$，即 $V_{max\_offset} = 0.15 V_t$）。

#### 2.3.2. 控制暂停与重置逻辑 (新增)
在定速控制运行期间，如果因为出现加速干扰（如人踩油门或下坡）导致控制被暂停（进入待命状态）：
* 当车速回落并重新切回“在控”状态时，必须**立即重置算法状态**，令阶段步数归零并初始化虚拟目标车速：
  $$i \leftarrow 1, \quad V_1 \leftarrow V_t \tag{7}$$
* 禁止保留并继承暂停前累加的历史虚拟目标车速 $V_i$，防止恢复控制时造成车辆突兀加速。

## 3. 加速干扰处理

CC 系统在控制中存在加速干扰，式（5）中的限制条件 $v_i(t) < V_t$ 就是为了避免加速干扰而设。
1. **人工加速与下坡**：若出现人工加速或下坡加速干扰，上述条件不满足时，控制暂停，维持待命，等待车速下降并满足该条件后，再重新初始化并进入控制。
2. **主动减速限制**：如果系统无法区分干扰是人工加速还是下坡，CC 系统将只进行加速扭矩控制，不实现主动制动减速控制。
